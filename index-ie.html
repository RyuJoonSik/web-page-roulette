자바스크립트 룰렛

    소개

    목표

    자바스크립트의 여러 프레임워크 학습에 들어가기 앞서 내용을 복습하고, 개발 도구(Babel, Webpack) 사용법 익히기.

    주제

    친구들과 만나면 어디로 갈지 단 번에 정하지 못해 대신 정해줄 방법이 있으면 좋을 것 같아 간단한 룰렛을 만들기로 하였습니다.

    결과물

    개발 환경

    브라우저

    • IE11
    • Chrome

    언어

    • HTML
    • CSS
    • JS

    에디터

    • VSC

    개발 도구

    • ESLint
    • Prettier

    개발 과정

    순서

      • HTML
      • CSS
      • JS
      • 바벨
      • Node.js 서버
      • 웹팩

    HTML

    의미론적 요소(Semantic element)
    article 요소와 section 요소를 사용하였습니다.
    접근성(Accessibility)
    canvas 태그에 대체 텍스트를 삽입하였습니다.
    콘텐츠 구분을 위해 구분선, 밑줄을 추가하였습니다.
    텍스트와 배경의 명도 대비는 4.5:1 이상으로 조정하였습니다.
    키보드 이동만으로도 사용할 수 있도록 하였습니다.
    검색 엔진 최적화(SEO)
    관련 메타 태그를 작성하였습니다.
    파비콘을 추가하였습니다.
    robot.txt 파일을 작성해 보았습니다.
    호환성(Cross Browsing)
    IE11와 크롬에서 공통으로 지원하는 태그와 속성을 사용하였습니다.

    CSS

    방법론(Methodology)
    BEM 방법론을 사용하였습니다.
    반응형(Responsive)
    CSS의 미디어 쿼리를 사용하였고, Pixel ratio(픽셀 비율)에 따라 캔버스가 렌더링 되도록 하였습니다.

    JS

    IE11에서 지원하는 함수와 ES5로 작성한 뒤 크롬에서 지원하는 함수와 ES6+ 이상으로 다시 작성하였습니다.

    변수 선언
    var, const, let의 특징을 구별하여 사용하였습니다.
    비동기 통신(AJAX)
    IE11에서는 XMLHttpRequest, 크롬에서는 fetch()를 사용하였습니다.
    편의성
    크롬에서는 템플릿 리터럴, 구조 분해 할당을 사용하였습니다.
    DOM 제어
    첫 초기화에만 innerHTML을, 이 후로는 객체를 생성하여 추가하였습니다.
    클릭 이벤트
    버블링을 통해 이벤트가 해당 요소에 전파되었을 경우에만 핸들러가 실행되도록 하였습니다.
    데이터 타입 검증
    메서드의 매개변수로 넘어온 인자의 타입을 검증하는 함수를 추가하였습니다.
    디자인 패턴
    싱글톤, 네임스페이스, 모듈 패턴을 사용해 보았습니다.
    싱글톤은 클로져를 이해하는 데 도움이 되었습니다.
    크롬에서는 ES6의 모듈(import, export)을 사용하였습니다.
    화살표 함수
    편의성보다 특징을 이해하는 데 중점을 두었습니다.
    private 접근자
    외부에서 접근할 필요가 없는 메서드와 필드는 private로 제한하였습니다.

    바벨

    • ES6+ 문법으로 작성된 코드를 ES5로 변환시켜 주었습니다.
    • IE11(브라우저)에서 지원하지 않는 함수들은 기존에 작성해 둔 ES5로 폴리필을 적용시켜 주었습니다.
    • 바벨 폴리필을 추가 설치하여 사용하였습니다.

    Node.js 서버

    Node.js 기본 모듈을 사용하여 간단한 서버를 만들어 보았습니다.

    바벨을 통해 변환된 코드 중 require 함수가 사용된 것을 보고, Node.js 환경을 통해 무난히 실행될 것이란 예상에 서버를 만들어 본 것이었지만 응답받은 파일은 결국 client-side(브라우저)에서 실행된 다는 사실을 이후에 알게 되었습니다.

    웹팩

    웹팩을 통해 분산되어 있는 외부 모듈들을 하나의 파일로 통합시켜 주었습니다. 바벨 로더를 통해 코드를 변환 시켜주는 작업도 함께해 주었습니다.

    마무리

    점검

    레퍼런스 사이트와 크롬 개발자 도구를 통해 불필요하거나 지양하는 코드를 수정하였습니다.
